# Prayukti-vLAB: Application Overview & Technical Documentation

## 1. Introduction & Vision
**Prayukti-vLAB** (Virtual Laboratory) is a premium, high-fidelity educational ecosystem designed to democratize access to laboratory resources for engineering students. Built primarily for **Madan Mohan Malaviya University of Technology (MMMUT)**, it bridges the gap between theoretical concepts and practical application through interactive, deterministic simulations.

The platform provides a "glamorous" yet rigorous learning environment where students can design, simulate, and verify experiments in key technical subjects, accessible from anywhere via a web browser.

## 2. Scope & Features

### Core Subjects
*   **Computer Networks (CN)**: Interactive visualization of protocols (OSI, CSMA/CD, Token Ring, Sliding Window).
*   **Object-Oriented Programming (OOPS)**: Concepts of classes, objects, and memory management.
*   **Digital Logic Design (DLD)**: A full-fledged circuit simulator for logic gates and combinational circuits.
*   **Database Management Systems (DBMS)**: Interactive SQL query execution and schema design (In Development).

### Key Features
*   **Interactive Simulation Engine**:
    *   **CN**: Procedural SVG animations driven by mathematical models.
    *   **DLD**: Drag-and-drop circuit builder with real-time state propagation and truth table generation.
*   **Visual Learning**: High-quality "Glassmorphism" UI with vibrant animations to make abstract concepts tangible.
*   **Assessment System**:
    *   Pre-lab and Post-lab quizzes (MCQs).
    *   Automated observation tables for collecting simulation data.
*   **Role-Based Access**:
    *   **Student Dashboard**: View enrolled subjects, track progress, and perform experiments.
    *   **Teacher/Admin Dashboard**: Manage subjects, monitor student performance, and content updates.
*   **University Integration**:
    *   Restricted access to institution domains (e.g., `@mmmut.ac.in`).
    *   Official branding and academic rigor.

## 3. Technology Stack

### Frontend (User Interface)
*   **Framework**: **Next.js 15** (App Router) for server-side rendering and routing.
*   **Language**: **TypeScript** (Strict mode) for type safety.
*   **Styling**: **Tailwind CSS** with **Shadcn/UI** (Radix UI primitives).
*   **State Management**: **Zustand** for global client state.
*   **Visualization & Graphics**:
    *   **Framer Motion**: For smooth, complex UI animations.
    *   **@xyflow/react (React Flow)**: For node-based editors (DLD Circuit Builder).
    *   **Recharts**: For plotting graphs and data visualization.
    *   **Spline**: For 3D interactive elements.
    *   **Lucide React**: For iconography.

### Backend (API & Logic)
*   **Runtime**: **Node.js** with **Express.js**.
*   **Language**: **TypeScript**.
*   **Database**: **MongoDB** with **Mongoose ODM**.
*   **Validation**: **Zod** for schema validation.
*   **Authentication**: Custom JWT-based auth with secure HttpOnly cookies.
*   **Security**: Bcrypt for password hashing.
*   **Communication**: RESTful API architecture.

### DevOps & Tooling
*   **Monorepo**: Managed by **TurboRepo** for efficient build and caching.
*   **Linting/Formatting**: ESLint, Prettier.
*   **Package Manager**: npm/yarn/pnpm (implied).

## 5. User Journey
1.  **Authentication**: Secure login (Student/Teacher) featuring domain validation.
2.  **Dashboard**: Landing area to select the specific Laboratory (Subject).
3.  **Experiment Selection**: List of practicals (e.g., "Implement CSMA/CD Protocol").
4.  **Learning Phase**: Structured tabs for **Aim**, **Theory**, and **Procedure**.
5.  **Simulation Phase**:
    *   User configures parameters (e.g., packet size, propagation delay).
    *   User interacts with component (e.g., wiring gates, clicking "Transmit").
    *   System provides visual feedback (animations, logs, graphs).
6.  **Analysis**: User records observations in the automated table.
7.  **Submission**: Results are saved (Future: Gradebook integration).

## 6. Future Scope & Roadmap
*   **Oditor.ai Integration**: A planned "Oditor Beta Platform" module incorporating Sandbox Manager, Flow Discovery, and Policy Engine for advanced network security and auditing simulations.
*   **Multiplayer Labs**: Real-time collaborative experiments using WebSockets (Socket.io) for "Pair Programming" style labs.
*   **AI Tutor (Neelam)**: Context-aware AI assistant to answer theory questions and debug student circuits in real-time.
*   **Code Execution Playground**: Sandboxed environment (WebAssembly) for safely running C++/Java code in OOPS/DBMS labs.
*   **Mobile Companion**: Optimized mobile view or PWA for studying theory and viewing results on the go.
*   **LMS Integration**: LTI support to sync grades directly with university ERP systems.
